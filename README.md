# № 1 Эксперименты с параллелизмом в Python: сравнение однопоточного, многопоточного и многопроцессорного подходов.
### CPU-bound задания
| Подход | Время |
|---|---|
| Однопоточность  |     2.22 с |
| Многопоточность   |   2.18 с |
| Многопроцессорность | 1.13 c |
## Вывод: По результатом было полученно, при CPU-bound задачах многопроцессорность выполняет вычислительные задания быстрее, чем однопоточность примерно в 2 раз

### I/O-bound задания
| Подход | Время |
|---|---|
| Однопоточность  |     9.59 с |
| Многопоточность   |   2.84 с |
| Многопроцессорность | 3.12 с |

## Вывод:  Для I/O-операций многопоточность и многопроцессорность дают схожий результат, но потоки затрачивают меньше памяти

## Выводы

1. CPU-bound:  Эксперимент показал, что однопоточность и многопоточности схожи во времени выполнении разница в 0.02 секунды, а многопроцессорность показала себя идеально и быстрее в 2 раза.
2. I/O-bound:  Многопоточность и многопроцессорность выдали близкие значения, но в I/O-bound заданиях проще будет использовать так как потоки затрачивают меньше памяти, чем многопроцессорность.

# № 2 Эксперименты с параллелизмом в Python: асинхронное программирование, синхронное программирование при запросе к внешнемму API 50 раз.

### I/O-bound задания
| Подход | Время |
|---|---|
| Асинхронный без семафора 50/50 успешных запросов |   1.31  с |
| Асинхронный с семафором 50/50 успешных запросов  |   11.68 с |
| Синхронный 50/50 успешных запросов | 153.53 с |

## Выводы 
1. При использование семафора мы ограничиваем, нагрузку на сервер, но при этом увеличиваем время выполнения, эксперимент показал что асинхронный подход с семафором выполняется дольше чем без семафора в 11 раз почти, такая большая разница объясняется тем что в эксперименте без семафора запросы ушли все одновременно, а у эксперимента с семафором там было ограниченное количество одноверменных запросов, но с меньшей нагрузкой на сервер.
2. Эксперимент показал что асихронный подход с aiohttp сессией быстрее почти в 13 раз, чем синхронный с requests это можно обяснить тем, что поток перключается между задачами,а не ждет ответа.
